# DevDenOS Documentation Overview
Welcome to DevDenOS documentation! This is an overview of what the documentation
will provide, and how it will be structured.

Documentation is handled by primarily Amethsyt (mysztikk), if there are
questions or any problems, she should be notified.

## What does documentation provide?
The documentation provides (or attempts to provide) thorough reference so
anybody can start programming quick on anything. The documentation also provides
specific specifications and principles that should be followed on the project to
help maintain the code base. Lastly, the documentation provides tons of external
resources that can help somebody who may be unfamiliar with Operating System
development start somewhere and gain enough understanding to contribute.

## How is the documentation structured?
Documentation is seperated into several specific categories. Specific categories
may share pages, but the idea of each category is to provide a different look on
the project depending on the reader's needs.

The categories are as follows:
- Introduction to DevDenOS (A brief guide on the internal workings, where to
                            start, and important concepts. This is page one!)
- User's Guide to DevDenOS (For users of the OS, and userland programming)
- Programmer's Guide to DevDenOS (Style, principles of programming, conventions,
                                  and other important reference information for
                                  someone programming on the project)
- DevDenOS Architecture (A thorough write-up of how the operating system works
                         internally)
- In-line documentation (Code comments that provide quick reference for using,
                         modifying, or reusing actual code in the code base)
- Reference documentation (Every function's behavior and important information
                           documented aggressively)
- File documentation (Every file documented specifically, with external learning
                      resources and specific advice about contributing to or
                      using code from the file, and any particular information
                      about it)
- Miscellaneous (Any information that didn't fit neatly into any of the above
                 categories)

Documentation is also provided in three particular forms. Some of the
documentation (specifically that which falls under the aforementioned "In-line
documentation category") is provided IN the code base, for the fastest, quickest
reference for programmers. All of the non in-line documentation however, is
available in markdown format (for viewing in the terminal) and website format
(for the best experience).
